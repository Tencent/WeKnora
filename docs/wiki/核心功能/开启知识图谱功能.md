# 开启知识图谱功能

本文档介绍如何在 WeKnora 中启用并验证知识图谱功能，支持 **Apache AGE**（推荐）和 **Neo4j** 两种后端。

## 前置条件

- 已完成 WeKnora 后端与前端的基础部署
- 具备可用的 Docker/Docker Compose 运行环境
- **AGE 模式**：使用已有的 PostgreSQL 数据库（无需额外服务）
- **Neo4j 模式**：本地或远端可访问的 Neo4j 服务

> 快速入门参见 [知识图谱](知识图谱.md)

## 步骤一：选择后端并配置环境变量

在项目根目录的 `.env` 文件中配置：

### AGE 模式（推荐）

```bash
GRAPH_DRIVER=age
AGE_ENABLE=true
AGE_GRAPH_NAME=weknora_kg
```

- 部署时无需额外启动图数据库容器，AGE 作为 PostgreSQL 扩展运行。
- 首次使用需初始化 AGE 扩展：
```bash
docker exec -i weknora-postgres psql -U postgres -d WeKnora < migrations/init-age.sql
```

### Neo4j 模式（向后兼容）

```bash
# 不设置 GRAPH_DRIVER 或设置为 neo4j
GRAPH_DRIVER=neo4j
NEO4J_ENABLE=true
NEO4J_URI=bolt://neo4j:7687
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=your_strong_password
```

- `NEO4J_URI` 中的 `neo4j` 为 docker-compose 服务名，如使用外部实例请替换为实际地址

## 步骤二：启动图数据库服务

### AGE 模式
无需额外操作，应用启动时会自动在 PostgreSQL 中创建图。

### Neo4j 模式
```bash
docker-compose --profile neo4j up -d
```

验证：
```bash
docker ps | grep neo4j
```

> 开发环境中 Neo4j 的启动方式参见 [开发指南](../开发部署/开发指南.md)

## 步骤三：重启 WeKnora 服务

```bash
make stop && make start
# 或者
docker compose up -d --build
```

确保后端日志中出现图数据库初始化成功的提示（AGE: `Using Apache AGE as graph database`，Neo4j: `Using Neo4j as graph database`）。

## 步骤四：在前端启用实体/关系抽取

1. 登录 WeKnora 前端管理页面
2. 打开「知识库设置」或创建新的知识库
3. 勾选「启用实体抽取」与「启用关系抽取」开关
4. 根据界面提示补充所需的 LLM、回调或模型参数（若有）

保存后，系统会在文档入库阶段自动触发实体与关系抽取任务。

## 步骤五：验证知识图谱

### AGE 模式

连接 PostgreSQL 查询图数据：
```sql
SET search_path = ag_catalog, public;
SELECT * FROM cypher('weknora_kg', $$ MATCH (n) RETURN n LIMIT 50 $$) AS (n agtype);
```

### Neo4j 模式

1. 访问 `http://localhost:7474`（或对应主机/端口）
2. 使用 `.env` 中的账号密码登录
3. 执行 `MATCH (n) RETURN n LIMIT 50;` 检查是否有新节点/关系

### WeKnora 界面

在知识库或对话页面中上传文档后，前端应展示图谱可视化入口；对话时系统会自动根据意图查询图谱并返回补充信息。

## 常见问题排查

- **无法连接图数据库**：确认 `GRAPH_DRIVER` 设置正确、对应的数据库服务已启动、环境变量配置完整
- **AGE 扩展未加载**：检查 PostgreSQL 容器是否已安装 AGE 扩展，确认已执行 `docker/init-age.sql`
- **未生成节点**：确认知识库已开启实体/关系抽取，且上传的文档已完成解析；查看后端日志中是否有抽取任务异常
- **Neo4j 查询无结果**：尝试在 Neo4j 控制台执行 `CALL db.schema.visualization;` 查看 schema 是否存在，必要时重新导入文档

> 更多排查建议参见 [常见问题](../运维排障/常见问题.md)

完成以上步骤后，知识图谱功能即成功启用，可结合 RAG 及 Agent 流程提升问答质量。
